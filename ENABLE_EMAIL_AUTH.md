## 🎯 How It Works Now

### Sign Up Flow
```
1. User fills signup form
   ↓
2. App calls Supabase signUp()
   ↓
3. Supabase creates user in auth.users
   ↓
4. Trigger creates profile in public.profiles
   ↓
5. User gets unique UUID
   ↓
6. Session created automatically
   ↓
7. User redirected to dashboard
```

### Login Flow
```
1. User enters email/password
   ↓
2. App calls Supabase signInWithPassword()
   ↓
3. Supabase verifies credentials
   ↓
4. Session token generated
   ↓
5. User data loaded
   ↓
6. User redirected to dashboard
```

### User ID Storage
- **User ID**: UUID generated by Supabase (e.g., `a1b2c3d4-e5f6-...`)
- **Stored in**: `auth.users` table
- **Profile**: Automatically created in `public.profiles`
- **Session**: Managed by Supabase (cookies)

## 🐛 Troubleshooting

### Issue: "User already exists"
**Cause**: Email already registered in Supabase
**Solution**: 
1. Try logging in instead, OR
2. Use different email, OR
3. Delete user from Supabase Dashboard

### Issue: "Invalid login credentials"
**Cause**: Wrong email or password
**Solution**: 
1. Check email spelling
2. Check password (case-sensitive)
3. Try password reset (future feature)

### Issue: "Email not confirmed"
**Cause**: Email confirmation is enabled
**Solution**: 
1. Check email for confirmation link, OR
2. Disable email confirmation in Supabase settings

### Issue: "Session expired"
**Cause**: Session timeout
**Solution**: Login again

### Issue: "Can't see user in Supabase"
**Cause**: Signup failed silently
**Solution**: 
1. Check browser console for errors
2. Verify Supabase URL in .env.local
3. Check Supabase keys are correct
